## 4. MVP Feature Set for Week 1

Only build or verify these in the first week.

- Guest network control works end to end.
- Device isolation / blocking works end to end.
- Alerts appear in the app with a clear reason.
- Cyber findings can be recorded and updated.
- Basic report output can be generated.

Optional only if time allows:

- GNSS / GPS anomaly v1.
- Edge exposure checks for router / VPN / firewall.

## 5. Success Criteria

The pilot is viable only if the owners agree these are true.

- We can see every major onboard device.
- We can block or isolate a device when needed.
- Alerts are understandable without a technical explanation.
- Weekly reporting is useful to the owner.
- The setup does not require constant manual intervention.

## 6. What We Are Not Building Yet

- Full OT protocol analytics.
- Full compliance automation packs.
- Deep AI detection pipelines.
- Large chart-heavy dashboards.
- Any feature that needs a long integration project before pilot value.

## 7. Pilot Risks To Watch

- Missing device inventory.
- Router access limitations.
- Too many noisy alerts.
- Unclear responsibility for vendor-side fixes.
- Pilot owners expecting a finished enterprise product too early.

## 8. Week 1 Output

By the end of week one we should have:

- A signed-off pilot scope.
- A completed hardware inventory for both boats.
- A locked MVP feature list.
- A list of required data inputs.
- A simple success-metrics sheet.
